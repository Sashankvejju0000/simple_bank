# Bank Management System Backend (FastAPI Rebuild)

A production-ready Bank Management System Backend REST API rebuilt using **Python**, **FastAPI**, and **SQLAlchemy**. It connects to your MySQL database `bankdb` and features JWT authentication, role-based authorization, request input validation, database transactions, and automatic trigger-based balance history logging.

## Technical Highlights
- **FastAPI Core**: Automatic documentation generation, type hints, and request validation via Pydantic.
- **SQLAlchemy ORM**: Connects directly to the existing database schema, utilizing robust connection-pool pooling.
- **ACID Transactions**: Money transfers run on isolation locks (`.with_for_update()`) inside database transaction blocks, eliminating race conditions.
- **Automatic Audit Trail**: Changes to any account balance automatically log to the `account_audit` table via your database trigger.
- **Standardized Error Formatting**: Central exception handlers trap all Joi-equivalent Pydantic, SQLAlchemy database, and HTTP errors into standardized JSON responses.

## Getting Started

### Prerequisites
- Python 3.10+
- MySQL Server (>= 8.0)

### 1. Database Configuration
Ensure your local MySQL server is running and the database tables and trigger are initialized. The credentials in your `.env` file should look like:
```ini
PORT=3008
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=root
DB_PASSWORD=sashank
DB_NAME=bankdb

JWT_SECRET=super_secret_bank_key_12345
```

### 2. Setup Virtual Environment & Install Dependencies
Create a virtual environment (optional but recommended):
```bash
python -m venv venv
venv\Scripts\activate      # On Windows
source venv/bin/activate    # On macOS/Linux
```

Install the required Python packages:
```bash
pip install -r requirements.txt
```

### 3. Running the Server
Run the development server using Uvicorn (includes live reload):
```bash
uvicorn app.main:app --reload --port 3008
```

The API will be live on `http://localhost:3008`.

## API Summary

### Authentication
- `POST /api/auth/register` - Create a Customer profile and login credentials.
- `POST /api/auth/login` - Obtain a JWT bearer token.

### Customers
- `GET /api/customers` - List all customer profiles (Admin only).
- `GET /api/customers/:id` - Retrieve profile info (Admin or Profile Owner).
- `POST /api/customers` - Create customer profiles (Admin only).
- `PUT /api/customers/:id` - Edit customer fields (Admin or Profile Owner).
- `DELETE /api/customers/:id` - Delete customer profiles (Admin only).

### Accounts
- `GET /api/accounts` - List all bank accounts (Admin only).
- `GET /api/accounts/:accountNo` - Retrieve account balance details (Admin or Account Owner).
- `POST /api/accounts` - Create new Savings/Checking account (Admin or Owner).
- `PUT /api/accounts/:accountNo` - Update account details (Admin only).
- `DELETE /api/accounts/:accountNo` - Remove account (Admin only).

### Transactions (under accounts router)
- `POST /api/accounts/deposit` - Deposit money into an account (Admin or Owner).
- `POST /api/accounts/withdraw` - Withdraw money from an account (Admin or Owner).
- `POST /api/accounts/transfer` - Transfer money to another account (Admin or Source Owner).
